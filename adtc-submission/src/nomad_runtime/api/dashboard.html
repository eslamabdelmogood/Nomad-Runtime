Plugin Architecture — every inference engine is a peer plugin discovered automatically from the plugins/ directory. Adding TensorRT, ExecuTorch, or any future engine means creating one new file. The router, API, and dashboard require zero changes.

How to add a new engine

Three methods override the required InferencePlugin contract. Zero files in the router, registry, or API change. 

from plugin_base import InferencePlugin, PluginCapabilities, \
    PluginHealthStatus, PluginInferenceResult

class TensorRTPlugin(InferencePlugin):
    engine_id = "tensorrt"          # matches MODEL_REGISTRY engine_id

    def capabilities(self) -> PluginCapabilities:
        return PluginCapabilities(engine_name="TensorRT",
                                  requires_gpu=True, typical_formats=["engine"])

    def health_check(self) -> PluginHealthStatus:
        # is TensorRT runtime available?
        ...

    def infer(self, model_id, prompt, system_prompt="",
              max_tokens=512, temperature=0.3) -> PluginInferenceResult:
        # call TensorRT — never raise, always return PluginInferenceResult
        ...

Save as plugins/tensorrt_plugin.py — it appears here on next restart.

Step 1 — Start Nomad Runtime API

The API server bridges this dashboard to the device monitor and inference engine.

cd nomad-runtime/api
python server.py

Runs on http://localhost:8765

Step 2 — Install Ollama & pull models

Ollama runs models locally with no cloud dependency.

# Install (Linux)
curl -fsSL https://ollama.com/install.sh | sh

# Nomad mode (~2.2 GB RAM)
ollama pull phi3:mini

# Workhorse mode (~3.8 GB RAM)
ollama pull llama3.2:3b

# Start server
ollama serve
Step 3 — Connect Autex

Add this to your Autex Next.js app to replace the Gemini API call with Nomad Runtime:

// src/lib/nomad-client.ts
const NOMAD_API = 'http://localhost:8765';

export async function nomadChat(prompt: string, systemPrompt?: string) {
  const res = await fetch(`${NOMAD_API}/chat`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ prompt, system_prompt: systemPrompt }),
  });
  return res.json();
  // Returns: { mode, model, response_text, latency_ms, ... }
}

Nomad Runtime auto-selects the best model. Autex needs zero changes to its UI.
